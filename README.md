# Booking Service

Демонстрационный сервис для управления бронированиями с асинхронной фоновой обработкой задач (Celery), реализованный на базе FastAPI и SQLAlchemy 2.0 (Async).

## Установка и запуск

### Требования

* Docker и Docker Compose
* UV (только для разработки и тестирования)
* Python 3.13+ (только для разработки и тестирования)

### Быстрый старт (Docker Compose)

1. Скопируйте шаблон окружения:

   ```bash
   cp .env.example .env
   ```

   Для быстрого запуска изменять файл `.env` не требуется.

2. Соберите образы и запустите сервисы (API, Worker, PostgreSQL, Redis)

    ```bash
    docker-compose up -d
    ```

    После запуска документация Swagger доступна по адресу: `http://localhost:8000/docs`

### Локальная разработка

1. Установите пакетный менеджер UV:

    ```bash
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```

2. Установите зависимости (включая dev):

    ```bash
    uv sync
    ```

3. Запустите PostgreSQL (например, через Docker)
4. Примените миграции: `prestart.sh` (скрипт)
   или

   ```bash
   alembic upgrade head
   ```

   или (через Makefile)

   ```bash
   make migrate
   ```

5. Запустите сервер:

    ```bash
    uvicorn src.main:app --reload --host 127.0.0.1 --port 8000
    ```

    или (через Makefile)

    ```bash
    make dev
    ```

## Тестирование

Проект покрыт юнит- и интеграционными тестами с использованием pytest + coverage. Для запуска не требуется сопутствующая инфраструктура (Docker), достаточно только установить зависимости.

```bash
uv run pytest .
```

Альтернатива (через Makefile):

```bash
make test
```

## Технические решения и архитектура

### Технологический стек

* **Фреймворк:** FastAPI
* **База данных:** PostgreSQL + SQLAlchemy 2.0 (async)
* **Миграции:** Alembic
* **Валидация:** Pydantic v2
* **Фоновые задачи:** Celery + celery-aio-pool
* **Лимит запросов:** slowapi
* **Тестирование:** pytest + coverage
* **Логирование:** logging (json)
* **Пакетный менеджер:** UV
* **Контейнеризация:** Docker + Docker Compose
* **Дополнительные средства:**
  * форматирование кода: Black + isort
  * статический анализ типов: MyPy

### Технические решения

#### Выбор фреймворка: FastAPI

FastAPI легковесный, имеет нативную поддержку асинхронности, встроенное автодокументирование API (Swagger), удобную систему внедрения зависимостей, поддержку Pydantic.

#### Подход к асинхронным задачам и идемпотентности

Для фоновой обработки бронирований используется Celery с брокером/бэкендом на базе Redis. В качестве пула workers применён celery-aio-pool, что позволяет выполнять внутри задачи полноценный async-код без блокировки event loop Python (критично для асинхронной SQLAlchemy 2.0).

В случае возникновения ошибки в процессе обработки задачи выполняется retry. В случае повтора задачи возвращается ранее установленный статус.

В случае отмены задачи — та удаляется. Данная операция идемпотентна. (В ТЗ не указан отдельный статус для состояния отменённой задачи, а также HTTP-метод DELETE явно предполагает удаление, несмотря на концепцию операции отмены. В более лучшем случае — это должен был быть `POST /booking/{id}/cancel`.)

### Архитектурные решения

Проект организован в соответствии с луковой (onion) архитектурой для обеспечения слабой связанности компонентов:

* `domain/`: Бизнес-модели (сущности), DTO, фильтры и перечисления. Не зависит от инфраструктуры или фреймворков.
* `services/`: Бизнес-логика приложения
* `infrastructure/`: Реализация внешних зависимостей (репозитории PostgreSQL, Celery продюсер).
* `api/http/`: Внешний API (HTTP).

Некоторые зависимости в домене намеренно не были инвертированы ввиду избежания внесения лишней сложности в виде необходимости постоянной поддержки интерфейсов.

## Краткое описание API

* **POST** `/bookings` — Создание бронирования
* **GET** `/bookings` — Список всех бронирований (с постраничной пагинацией)
* **GET** `/bookings/{booking_id}` — Получение бронирования по ID
* **DELETE** `/bookings` — Отмена бронирования (идемпотентная операция)
